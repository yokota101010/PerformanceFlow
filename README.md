# PerformanceFlow - プロジェクトマスタ管理 (F01)

本プロジェクトは、システム内のプロジェクト情報を管理する「F01 プロジェクトマスタ管理」機能を実装したWebフロントエンドSPAアプリケーションです。

---

## 🚀 特徴 (Key Features)
* **クリーンアーキテクチャ設計**: ドメイン層、アプリケーション層、インフラストラクチャ層の3層構造に分離し、依存性を一方向に制御。
* **データ永続化**: ブラウザの `LocalStorage` を使用したローカル完結型の永続化。
* **一意性バリデーション**: プロジェクト名の一意性をアプリケーション層で厳格に検証し、重複登録を防止。
* **ID自動採番 (`PJnnn`)**: 既存の最大ID値に `+1` したIDを自動採番（削除時の欠番は安全のため再利用しない）。
* **削除制限**: 案件データ（他集約）に紐づいているプロジェクトIDの削除を自動的にブロックし、参照整合性を維持。
* **入力値トリミング**: プロジェクト名の保存およびバリデーションの直前に、前後のスペース（全角・半角）を自動的に除去。
* **グラスモルフィズムUI**: 美しいダークテーマに調和する、半透明でモダンなカード型UIデザイン。

## 🏃 実行方法 (How to Run)

### 依存パッケージのインストール
```bash
npm install
```

### 開発用ローカルサーバーの起動
```bash
npm run dev
```
起動後、ブラウザで [http://localhost:5173](http://localhost:5173) を開いて動作を確認します。

### スタンドアローン形式でのビルドと実行 (推奨)
本プロジェクトは、Webサーバーを使わずに、ブラウザでHTMLを直接ダブルクリックするだけで動かせる完全なスタンドアローン形式でのビルドをサポートしています。
1. 以下のコマンドでビルドを実行します：
   ```bash
   npm run build
   ```
2. ビルド完了後、`dist/index.html` が単一ファイルとして生成されます。このファイルをブラウザで直接開く（ダブルクリックまたはドラッグ＆ドロップ）ことで、ローカルサーバーなしでアプリ全体の動作を確認することができます。

### 自動テストの実行
```bash
npm run test
```
Vitest によるユニットテストおよびUI結合テストを一括で実行します。

## 📂 プロジェクト構造
```text
src/
├── domain/
│   ├── models/                  # エンティティ定義 (Project.ts)
│   └── repositories/            # リポジトリインターフェース
├── application/
│   ├── usecases/                # ユースケース抽象定義 (DTO含む)
│   └── services/                # 具象ユースケースロジック (ProjectService.ts)
└── infrastructure/
    ├── persistence/             # LocalStorage & インメモリリポジトリ、レジストリ
    └── ui/                      # React コンポーネント (ProjectView, ProjectForm)
```
