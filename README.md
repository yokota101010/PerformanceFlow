# PerformanceFlow - プロジェクト・収支・工数統括管理システム (F01 - F11)

本プロジェクトは、IT開発における各種マスタ（プロジェクト、社員、発注先、要員、案件）、契約・実績データ（案件アサイン契約、パートナー発注、社員工数実績、その他経費）、および分析・ダッシュボード機能（業績・収支サマリ、要員別工数サマリ）を網羅した、完全ローカル完結型のWebフロントエンドSPAアプリケーションです。

---

## 🚀 特徴 (Key Features)

* **クリーンアーキテクチャ設計**: ドメイン層、アプリケーション層、インフラストラクチャ層の3層構造に分離し、依存性を一方向に制御。
* **データ永続化**: ブラウザの `LocalStorage` を使用したローカル完結型の永続化。リロード後もデータが維持され、初期シードデータの自動投入に対応。
* **動的原価算出＆結果整合性**: 各案件アサイン契約の「製造原価」は、社員工数加工費・パートナー発注額・その他経費の最新データから動的にリアルタイム集計・再計算。

## 📋 機能一覧 (Functionalities)

### 1. マスタ管理 (F01 - F05)
* **プロジェクトマスタ管理 (F01)**:
  * **一意性バリデーション**: プロジェクト名の一意性をアプリケーション層で検証し、重複登録を防止。
  * **ID自動採番 (`PJnnn`)**: 最大値に `+1` したIDを自動生成（欠番再利用なし）。
  * **削除制限**: 案件データ（他集約）に紐づくプロジェクトIDの削除をブロック。
* **社員マスタ管理 (F02)**:
  * **ID自動採番 (`EMPnnn`)**: 最大値に `+1` したIDを自動生成。
  * **同姓同名の許可**: 氏名重複での新規・更新を許容し、社員IDによって一意性を識別。
  * **単価の遡及再計算**: 社員単価更新時、工数実績レコードに紐づく「加工費」に対し最新単価を動的適用。
  * **削除制限**: 対象社員を参照する工数データが存在する限り削除をブロック。
* **発注先マスタ管理 (F03)**:
  * **ID自動採番 (`BPnnn`)**: 最大 `BP999` まで自動採番。
  * **一意性バリデーション**: 同一の発注先名（取引先名）での重複登録・更新をブロック。
  * **削除制限**: 所属する「要員」または「注文実績」からの参照が存在する限り削除をブロック。
  * **初期シード自動投入**: 「Ａソフトウェア」「Ｂエンジニアリング」を初回自動ロード。
* **要員マスタ管理 (F04)**:
  * **ID自動採番 (`MEMnnn`)**: 最大 `MEM999` まで自動採番。
  * **所属会社ID検証**: 発注先マスタに存在する発注先IDとの紐づけを検証。
  * **削除制限**: 注文明細または月別要員工数サマリから参照されている限り削除をブロック。
  * **初期シード自動投入**: 坂本龍馬、高杉晋作、西郷隆盛、勝海舟の4名を自動ロード。
* **案件管理 (F05)**:
  * **プロジェクト別自動採番 (`AJnnn`)**: 同一プロジェクト内の最大値に `+1` したIDを自動生成。
  * **同一プロジェクト内重複名禁止**: 同一プロジェクト内での同名案件をブロック。
  * **日付整合性バリデーション**: 開始日・終了日の形式検証および順序（開始日 <= 終了日）を保証。
  * **削除制限**: 案件作業明細（アサイン契約）から参照されている限り削除をブロック。

### 2. 契約・実績・収支管理 (F06 - F11)
* **案件作業契約（明細）管理 (F06)**:
  * **プロジェクト別自動採番 (`WKnnn`)**: プロジェクト単位で連番管理。
  * **終了日の自動導出**: 後続明細の開始日前日、または案件終了日を自動セット。
  * **期間整合性・隙間チェック**: 案件全体の開始日〜終了日を隙間なくカバーすることを保証し、期間重複を防止。
  * **原価・粗利・粗利率の自動計算**: 売上、製造原価、粗利、粗利率をリアルタイム導出。
* **発注先注文管理 (F07)**:
  * **注文ID自動採番 (`ORDnnn`)**: 発注契約および注文明細を一括管理。
  * **期間・所属バリデーション**: 作業契約期間外の年月登録を防止。要員の所属会社と発注先IDの一致を強制。
  * **カスケード削除**: 親発注データの削除時に紐づく注文明細を自動連携削除。
* **社員工数実績管理 (F08)**:
  * **加工費自動算出**: 「社員単価 × 作業時間 (h)」で加工費をリアルタイム算出。
  * **期間・重複バリデーション**: 作業契約期間外の入力や、同一「作業契約＋社員＋年月」の重複入力をブロック。
  * **稼働上限警告**: 同一社員の月間合計稼働時間が200時間を超過した場合にアラート表示。
* **その他経費管理 (F09)**:
  * **複合主キー管理**: 作業契約IDと行Noによる一意管理。
  * **製造原価へ即時反映**: 旅費交通費・会議費等の入力結果が案件アサインの製造原価に即座に影響。
* **業績・収支サマリ表示 (F10)**:
  * **全社＆プロジェクト別ダッシュボード**: 全社の売上合計・製造原価合計・粗利合計・全体粗利率をサマリカード表示。
  * **原価ブレイクダウン表示**: 製造原価の内訳（加工費、発注額、その他経費）を常時並列表示。
  * **赤字・低粗利率警告**: 粗利率マイナスの案件・アサイン契約をハイライト（赤字＆赤背景）表示。
  * **条件フィルタリング**: プロジェクト名や期間指定による動的絞り込み。
* **要員別月別工数サマリ表示 (F11)**:
  * **マトリクス表示**: 要員を行、年月を列とした全社の稼働状況グリッド表示。
  * **ライトバック自動同期**: 注文明細データに基づき、画面起動時に集計結果を自動書き戻し（ライトバック永続化）。
  * **オーバーアサイン警告**: 1.0人月を超過した月・要員に対して警告アイコン（⚠）および赤字太字アラートを表示。

## 🏃 実行方法 (How to Run)

### 依存パッケージのインストール
```bash
npm install
```

### 開発用ローカルサーバーの起動
```bash
npm run dev
```
起動後、ブラウザで [http://localhost:5173](http://localhost:5173) を開いて動作を確認します。

### スタンドアローン形式でのビルドと実行 (推奨)
本プロジェクトは、Webサーバーを使わずに、ブラウザでHTMLを直接ダブルクリックするだけで動かせる完全なスタンドアローン形式でのビルドをサポートしています。
1. 以下のコマンドでビルドを実行します：
   ```bash
   npm run build
   ```
2. ビルド完了後、`dist/index.html` が単一ファイルとして生成されます。このファイルをブラウザで直接開く（ダブルクリックまたはドラッグ＆ドロップ）ことで、ローカルサーバーなしでアプリ全体の動作を確認することができます。

### 自動テストの実行
```bash
npm run test
```
Vitest によるユニットテストおよびUI結合テストを一括で実行します。
