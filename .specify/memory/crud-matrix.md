# システム全体鳥瞰図 (CRUD Matrix)

本書は、システムで扱う各ドメインデータモデル（[domain-model.md](file:///.specify/memory/domain-model.md)）と、アプリケーションが提供する主要機能（画面・ユースケース）との関係性を示したシステム全体の鳥瞰図（CRUD対応表）である。

本ドキュメントは、システム開発憲法（[constitution.md](file:///.specify/memory/constitution.md)）で定義されている **「単一トランザクション・単一集約 of 原則」** および **「Command系（変更系）画面の制約」** に基づき、ドメイン集約の境界と1対1で対応する機能（Command）構成として設計されています。

---

## CRUD マトリクス (CRUD Matrix)

各機能が各ドメインエンティティ（テーブル）に対して実行する操作（**C**: Create / **R**: Read / **U**: Update / **D**: Delete）の対応表です。

憲法に定める **「結果整合性の許容とプル型同期（オンデマンド再計算）」** およびドメインモデルの **外部キー（FK）制約の依存関係** に従い、機能構成と操作権限を定義しています。
