# Feature Specification: 業績・収支サマリ表示 (financial-summary)

**Feature Branch**: `010-financial-summary`

**Created**: 2026-07-19

**Status**: Draft

**Input**: User description: "F10の開発を行う"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 全社収支サマリカードの表示 (Priority: P1) 🎯 MVP

システム全体の売上合計、製造原価合計、粗利合計、および全体の粗利率を視覚的なサマリカードとして画面上部に表示する。

**Why this priority**:
経営者やマネージャーがシステムを開いた際、全社またはプロジェクト全体の収支概況を一目で把握できるようにするための最重要MVP機能。

**Independent Test**:
LocalStorage に初期シードデータがロードされた状態で本画面を開いた際、全アサイン明細の「売上」「製造原価（加工費＋発注額＋その他経費）」が集計され、画面上部のサマリカードに正しい値で表示されること。

**Acceptance Scenarios**:

1. **Given** アプリケーション内に初期シードデータ（WK001〜WK004）が登録されているとき、
   **When** ユーザーが「業績・収支サマリ」画面を開いたとき、
   **Then** 画面上部に以下の合計金額が表示されること。
     - 売上合計: `18,800,000円` (8,000,000 + 8,000,000 + 1,400,000 + 1,400,000)
     - 製造原価合計: `15,437,000円` (5,242,000 + 5,215,000 + 2,490,000 + 2,490,000)
     - 粗利合計: `3,363,000円`
     - 全体粗利率: `18%` (3,363,000 ÷ 18,800,000)

### User Story 2 - プロジェクト・案件別収支明細テーブルの表示 (Priority: P1)

プロジェクト・案件、および作業契約（アサイン）ごとの収支内訳をグリッドテーブルで一覧表示する。

**Why this priority**:
どのプロジェクトや案件が黒字であり、どこに原価がかかっているのかのボトルネックを特定するため。

**Independent Test**:
収支明細テーブルにおいて、プロジェクト名や案件名がグループ化またはネストされて表示され、それぞれの売上、製造原価（内訳含む）、粗利、粗利率が正しく計算されて表示されていること。

**Acceptance Scenarios**:

1. **Given** ユーザーがサマリ画面を表示しているとき、
   **When** 収支明細テーブルを確認したとき、
   **Then** 各アサイン契約（WK001など）の行に、売上、製造原価、粗利、粗利率に加えて、製造原価の独立した内訳カラム（加工費、発注額、その他経費）が常時並んで表示されること。
2. **Given** 収支明細テーブルが表示されているとき、
   **When** 粗利率がマイナスの行（例: WK003, WK004は粗利率-78%）が存在するとき、
   **Then** 該当行の粗利率・粗利額のテキストが赤字で表示され、かつ行全体の背景色が薄い赤色（警告色）で強調表示されること。

### User Story 3 - 条件フィルタリング機能 (Priority: P2)

プロジェクト名や契約期間で表示データを絞り込む。

**Why this priority**:
データ件数が増加した際に、特定のプロジェクトや期間にフォーカスして収支を分析できるようにするため。

**Independent Test**:
検索フォームにプロジェクト名の一部を入力、または期間を指定して検索した際、サマリカードおよび明細テーブルが即座にフィルタリング再計算されて更新されること。

**Acceptance Scenarios**:

1. **Given** プロジェクトが複数登録されている状態で、
   **When** ユーザーがフィルターで「次世代基幹」と入力して適用したとき、
   **Then** 「次世代基幹システム開発プロジェクト」に紐づくアサイン明細のみが抽出され、合計サマリカードおよびテーブルの数値がその範囲に絞り込まれて再集計されること。
2. **Given** 期間フィルターで「2026-08」〜「2026-09」を指定したとき、
   **When** フィルターを適用したとき、
   **Then** 指定期間内に重なりを持つアサイン契約のみが抽出され、再集計されること。

## Requirements *(mandatory)*

### ユビキタス言語辞書

| 業務上の用語 | コード内の英語表記 | 説明 |
| :--- | :--- | :--- |
| 売上合計 | totalSales | 抽出された全アサイン契約の売上の合算 |
| 製造原価合計 | totalCost | 抽出された全アサイン契約の製造原価の合算 |
| 粗利合計 | totalGrossProfit | 売上合計 － 製造原価合計 |
| 全体粗利率 | overallGrossProfitRate | 粗利合計 ÷ 売上合計（パーセント表示） |
| 原価ブレイクダウン | costBreakdown | 製造原価を「加工費」「発注額」「その他経費」に分けた内訳情報 |

### Functional Requirements

- **FR-001**: システムは全アサイン明細の「売上」「製造原価」をリアルタイムに集計し、売上合計、製造原価合計、粗利合計、全体粗利率を画面上部に表示しなければならない。
- **FR-002**: システムは各アサイン明細に対して、売上、製造原価、粗利、粗利率をテーブル形式で一覧表示しなければならない。
- **FR-003**: 収支サマリの表示は「プロジェクト」または「アサイン契約」単位の一覧をメインとし、ユーザーはプロジェクト名（部分一致）および契約期間（開始月・終了月）で表示データをフィルタリングできなければならない。
- **FR-004**: 各項目の計算式は以下の通りとし、画面上に導出値を動的に表示しなければならない。
  - 売上 ＝ `契約工数` × `契約単価`
  - 製造原価 ＝ `工数加工費` ＋ `発注額` ＋ `その他経費`
  - 粗利 ＝ `売上` － `製造原価`
  - 粗利率 ＝ `粗利` ÷ `売上` (小数点以下第3位を四捨五入してパーセント表記)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 業績・収支サマリ画面を開いてから、全データのロード、集計計算、および画面描画が完了するまでの時間が 0.5秒以内であること。
- **SC-002**: プロジェクトマスタやアサイン契約、工数、経費、発注の各画面でデータが更新された後、本画面に戻った際に最新のデータが完全に同期反映されていること。

## Assumptions

- 本機能は参照専用（Query）画面であり、データの直接の新規登録や編集・削除機能は持たないものとする（データの修正はそれぞれのマスタまたは実績入力画面で行う）。
- 表示される金額はすべて日本円（円単位）とし、カンマ区切りでフォーマット表示する。
- データの集計はブラウザ側でオンデマンドに実行され、外部API通信は行わない（LocalStorage 完結）。
