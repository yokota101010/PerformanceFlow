# Quickstart & Verification Guide: 業績・収支サマリ表示 (financial-summary)

F10「業績・収支サマリ表示」の検証シナリオ、および実行手順を示すガイド。

## 1. 開発環境の起動
```bash
# ローカル開発サーバー（デバッグポータル）の起動
npm run dev
```
ブラウザで `http://localhost:5173/` にアクセスし、ナビゲーションから「その他経費入力」または「業績・収支サマリ」（追加されたタブ）が確認できることを検証します。

---

## 2. 受入検証シナリオ (Acceptance Verification Scenarios)

### シナリオ1: 全社収支サマリカードおよびテーブルのロード検証 (US1, US2)
1. 「収支サマリ」タブを開く。
2. 画面上部に、以下の合計収支カードが表示され、カンマ区切りでフォーマットされていることを確認する。
   - 売上合計: `18,800,000` 円
   - 製造原価合計: `15,437,000` 円
   - 粗利合計: `3,363,000` 円
   - 全体粗利率: `18%`
3. テーブルに以下の4つのアサイン行（WK001〜WK004）が一覧表示され、各列「売上」「工数加工費」「発注額」「その他経費」「製造原価」「粗利」「粗利率」が常時表示されていることを確認する。
4. テーブル内に以下の赤字（マイナス粗利率）警告行が存在し、赤字強調（粗利率・金額が赤色、かつ行全体の背景色が薄い赤色）されていることを確認する。
   - `WK003` (粗利: -1,090,000円, 粗利率: -78%)
   - `WK004` (粗利: -1,090,000円, 粗利率: -78%)

### シナリオ2: フィルター検索の検証 (US3)
1. フィルター検索フォームの「プロジェクト名」入力欄に「次世代基幹」と入力し、「適用」をクリックする。
2. テーブルに「次世代基幹システム開発プロジェクト」に属する行のみが表示されることを確認する。
3. 開始月フィルターに「2026-08」〜「2026-09」を指定して適用した際、期間内に重なりを持つアサイン契約のみが正しくフィルタリングされ、上部の合計サマリカード（売上、原価、粗利）が自動で再計算・更新されることを確認する。
