# Feature Specification: 要員別工数サマリ表示 (member-worktime-summary)

**Feature Branch**: `011-member-worktime-summary`

**Created**: 2026-07-19

**Status**: Draft

**Input**: User description: "F11の開発を行う"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 要員別月別合計工数の一覧表示 (Priority: P1) 🎯 MVP

要員ごと・年月ごとの合計工数をグリッド形式で一覧表示し、要員の空き状況やアサイン状況を可視化する。

**Why this priority**:
要員の稼働状況を把握し、アサイン不足やアサイン過多（オーバーアサイン）を迅速に発見するための最重要MVP機能。

**Independent Test**:
LocalStorage に初期シードデータがロードされた状態で「要員工数サマリ」画面を開いた際、各要員（MEM001〜MEM004）の年月ごとの合計工数が正しく表示されていること。

**Acceptance Scenarios**:

1. **Given** アプリケーション内に初期シードデータが登録されているとき、
   **When** ユーザーが「要員工数サマリ」画面を開いたとき、
   **Then** グリッド上に以下の要員別合計工数が表示されること。
     - 要員 `MEM001` (所属: パートナー社員): 2026-08から2026-11の各月で `0.8` 人月
     - 要員 `MEM002` (所属: パートナー社員): 2026-08から2026-11の各月で `0.5` 人月
     - 要員 `MEM003` (所属: パートナー社員): 2026-09および2026-10の各月で `1.0` 人月
     - 要員 `MEM004` (所属: パートナー社員): 2026-09および2026-10の各月で `0.6` 人月

---

### User Story 2 - 工数集計のオンデマンド同期（ライトバック） (Priority: P1)

注文明細（発注）のデータに基づいて、要員・年月ごとの合計工数を自動で集計し、`月別要員工数サマリ` テーブルへ書き戻し同期（ライトバック）する。

**Why this priority**:
合計工数データの一貫性を保ちつつ、憲法に準拠したプル型同期（ライトバック）を実現するため。

**Independent Test**:
新しく注文明細を登録、または変更した後に「要員工数サマリ」画面を開き、LocalStorage の `PF_MonthlyMemberWorkHoursSummaries` 内の合計工数が正しく再集計され、書き戻されていることを確認すること。

**Acceptance Scenarios**:

1. **Given** ユーザーが要員工数サマリ画面を表示したとき、
   **When** 画面起動（表示）時のタイミングで、最新の `注文明細` の `発注工数` の合計値が `月別要員工数サマリ` テーブルへ自動的に書き戻されること。
2. **Given** `月別要員工数サマリ` に書き戻されたとき、
   **When** データが LocalStorage (キー: `PF_MonthlyMemberWorkHoursSummaries`) に永続化保存されること。

---

### User Story 3 - オーバーアサイン警告表示 (Priority: P2)

同一年月において、特定の要員の合計工数が `1.0` 人月を超過している場合にビジュアル警告を表示する。

**Why this priority**:
1人のメンバーが複数のプロジェクトや案件に過剰にアサインされ、過重労働や計画遅延のリスクが発生していることをアラートするため。

**Independent Test**:
特定の要員の工数合計が `1.2` などの超過状態になっている月がある場合、そのセルの色が警告表記に変わっていること。

**Acceptance Scenarios**:

1. **Given** 注文明細の追加により、要員 `MEM001` の 2026-08月期の合計工数が `1.3` になったとき、
   **When** 要員工数サマリ画面を表示したとき、
   **Then** 超過した合計工数セルの文字が赤色かつ太字（警告色）になり、警告アイコン（⚠）が数値の横に表示されること。

## Requirements *(mandatory)*

### ユビキタス言語辞書

| 業務上の用語 | コード内の英語表記 | 説明 |
| :--- | :--- | :--- |
| 合計工数 | totalEffort | 特定の要員・年月における発注工数の合算値 |
| 月別要員工数サマリ | MonthlyMemberWorkHoursSummary | 要員ごと・年月ごとの工数を保持する集約 |
| オーバーアサイン | overAssignment | 合計工数が 1.0人月を超過している状態 |

### Functional Requirements

- **FR-001**: システムは画面起動時に `注文明細` から要員ID・年月ごとの `発注工数` を合計集計し、`月別要員工数サマリ` レポジトリへ保存・更新しなければならない。
- **FR-002**: システムは各要員に対して、年月ごとの合計工数をグリッド形式（要員を行、年月を列とするマトリクス表示）で一覧表示しなければならない。
- **FR-003**: システムは合計工数が `1.0` を超えている箇所に対し、自動的にオーバーアサインの警告状態として識別しなければならない。
- **FR-004**: 各要員の基本情報（氏名、所属会社名等）を要員マスタおよび発注先マスタから引き当てて表示しなければならない。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 画面ロード時に自動同期（ライトバック）が走り、描画完了するまでの時間が 0.5秒以内であること。
- **SC-002**: 注文明細データが変更された後、本サマリ画面を表示した際に即座に集計が再計算され、ライトバック永続化が自動で行われていること。

## Assumptions

- 工数実績（社員工数）は本サマリの対象外とし、発注契約（パートナー注文明細）の「発注工数」のみを集計対象とする（SoT定義に準拠）。
- 表示される年月は、現在登録されているすべての注文明細およびシードサマリデータに存在する最小の年月から最大の年月までの範囲をカバーする。
