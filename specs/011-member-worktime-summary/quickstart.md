# Quickstart & Verification Guide: 要員別工数サマリ表示 (member-worktime-summary)

F11「要員別工数サマリ表示」の検証シナリオ、および実行手順を示すガイド。

## 1. 開発環境の起動
```bash
npm run dev
```
ブラウザで `http://localhost:5173/` にアクセスし、ナビゲーションから「要員工数サマリ」タブが確認できることを検証します。

---

## 2. 受入検証シナリオ (Acceptance Verification Scenarios)

### シナリオ1: 初期データの自動集計・ライトバックおよびマトリクス表示検証 (US1, US2)
1. 「要員工数サマリ」タブを開く。
2. 画面起動と同時にバックグラウンドで `注文明細` からの自動集計が走り、LocalStorage の `PF_MonthlyMemberWorkHoursSummaries` 内にデータが自動的にライトバック保存されることを確認する。
3. 画面上に以下の要員ごとの年月別工数がマトリクス表示されることを確認する。
   - `MEM001` の 8月〜11月: 各月 `0.8`人月
   - `MEM002` の 8月〜11月: 各月 `0.5`人月
   - `MEM003` の 9月〜10月: 各月 `1.0`人月
   - `MEM004` の 9月〜10月: 各月 `0.6`人月

### シナリオ2: オーバーアサイン警告の表示検証 (US3)
1. 特定の要員に対して、同月内の合計発注工数が `1.0` 人月を超えるように、発注管理等でアサインを追加する（例: MEM001 の 2026-08月期の合計を `1.2` にする）。
2. 「要員工数サマリ」画面を開き、該当月セルの数値（例: `1.2`）の文字が **赤太字** になり、かつ数値の横に **警告アイコン（⚠）** が表示されることを確認する。
