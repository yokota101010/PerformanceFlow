# Data Model: F01 プロジェクトマスタ管理

本ドキュメントは、「F01 プロジェクトマスタ管理」機能で扱うドメインエンティティおよびそのビジネスルール、バリデーション仕様を定義する。

---

## 1. プロジェクト集約 (Project Aggregate)

### 👑 集約ルート (Aggregate Root): `Project`
本集約は「プロジェクト」単一のエンティティで構成される。外部マスタへの依存はなく、自己完結している。

### エンティティ定義

#### `Project` 型定義 (TypeScript)
憲法に定める「不変性の強制（`readonly` 修飾子）」に従い、以下のように定義する。

```typescript
export interface Project {
  readonly id: string;      // プロジェクトID (形式: PJnnn, 主キー)
  readonly name: string;    // プロジェクト名 (一意)
}
```

---

## 2. カラム定義とビジネスルール

### カラム定義

| 属性名 | 物理名 | 型 | PK | UQ | 必須 | 区分 | ビジネスルール・検証仕様 |
|:---|:---|:---|:---:|:---:|:---:|:---|:---|
| プロジェクトID | `id` | `VARCHAR(255)` | ✅ | | ✅ | 独立 | 形式：`PJnnn`<br>・`PJ` は固定文字列<br>・`nnn` は `001` から始まる3桁の連番（最大 `999`）<br>・システムによる自動採番（編集不可） |
| プロジェクト名 | `name` | `VARCHAR(255)` | | ✅ (UQ1) | ✅ | 独立 | ・前後の空白（全角・半角スペース）はトリミングされる<br>・トリミング後の長さは 1〜255 文字<br>・システム全体で一意（他プロジェクトとの重複不可） |

## 3. 状態遷移とライフサイクル

### ライフサイクルアクション
`Project` エンティティは状態値（ステータス）を持たず、ライフサイクルはデータストア（LocalStorage/メモリ）への登録・更新・削除に直結する。

1. **生成 (Create)**
   * トリミング済みの `name` の必須チェックおよび重複チェックを実行。
   * 問題がなければ自動採番（`PJ` + 最大値 `+1`）された `id` を持つ `Project` インスタンスを永続化ストレージに追加する。
2. **変更 (Update)**
   * 既存の `id` に紐づくエンティティに対して、新しい `name` のトリミング、必須チェック、および他レコードとの重複チェックを実行。
   * 検証をパスした場合、`name` を更新した新しい `Project` インスタンスを再保存する（`id` は不変）。
3. **消滅 (Delete - 物理削除)**
   * 削除対象の `id` を持つプロジェクトを、データストアから完全に削除（消滅）させる。
   * **事前チェック（参照整合性）**: 削除対象の `id` が `案件` リソース（`案件.プロジェクトID`）によって参照されている場合は、削除処理をブロックしてエラーを発生させなければならない。
