# Quickstart Validation Guide: F01 プロジェクトマスタ管理

本ドキュメントは、「F01 プロジェクトマスタ管理」機能の実装後、開発者が機能要件（CRUD、重複チェック、削除制限など）をローカル環境で検証するための手順および検証シナリオを定義する。

---

## 1. 準備と実行手順 (Setup & Execution)

### 前提条件 (Prerequisites)
* Node.js v18 以上
* npm パッケージマネージャ

### アプリケーションの起動
1. 依存パッケージのインストール：
   ```bash
   npm install
   ```
2. 開発用ローカルサーバーの起動：
   ```bash
   npm run dev
   ```
   * 起動後、ブラウザで [http://localhost:5173](http://localhost:5173)（または表示されたローカルURL）を開きます。
3. 自動テストの実行：
   ```bash
   npm run test
   ```

---

## 2. 受入検証シナリオ (Validation Scenarios)

### シナリオ1: アプリ初回起動時の初期データ自動投入検証
* **検証内容**: 永続化ストレージ（LocalStorage）が空の状態でアプリを初めて開いた際、シードデータが正しく投入されること。
* **手順**:
  1. ブラウザのデベロッパーツールを開き、「アプリケーション（Application）」タブから `LocalStorage` のデータを全消去（クリア）する。
  2. 画面をリロードする。
  3. プロジェクト一覧画面に、以下のプロジェクトが表示されていることを確認する。
     * **プロジェクトID**: `PJ001`
     * **プロジェクト名**: `次世代基幹システム開発プロジェクト`

### シナリオ2: 新規登録とID自動採番の検証
* **検証内容**: 新しいプロジェクトが自動採番（最大値 + 1）されたIDで追加されること。
* **手順**:
  1. プロジェクト一覧画面または登録フォームを開く。
  2. プロジェクト名に `新規製品開発プロジェクト` と入力し、「登録」ボタンをクリックする。
  3. 一覧画面に以下の通り追加されていることを確認する。
     * **プロジェクトID**: `PJ002`（`PJ001` の最大値 `1` に `+1` されたもの）
     * **プロジェクト名**: `新規製品開発プロジェクト`

### シナリオ3: 重複登録防止の検証
* **検証内容**: 同名のプロジェクト登録・更新がシステムによってブロックされること。
* **手順**:
  1. 登録フォームにて、登録済みの `次世代基幹システム開発プロジェクト` と同一の名前を入力し、「登録」をクリックする。
  2. 画面に「このプロジェクト名は既に登録されています」などのエラーメッセージが表示され、登録が拒否されることを確認する。
  3. 既存の `PJ002` の編集画面を開き、名前を `次世代基幹システム開発プロジェクト` に変更して「保存」をクリックする。
  4. 同様に重複エラーメッセージが表示され、更新が拒否されることを確認する。

### シナリオ4: 入力値トリミングの検証
* **検証内容**: 入力前後のスペースが自動的に削除されること。
* **手順**:
  1. 登録フォームで、プロジェクト名に `  新規製品開発プロジェクト改  `（前後にスペース）と入力して登録を実行する。
  2. 登録完了後、一覧画面に表示された名称が `新規製品開発プロジェクト改`（スペースなし）になっていることを確認する。
  3. 登録フォームで、スペースのみ（例: `  `）入力して登録を実行する。
  4. 「プロジェクト名は必須です」とエラーが表示され、登録できないことを確認する。

### シナリオ5: 削除制約の検証
* **検証内容**: 案件データが紐づいているプロジェクトは削除が拒否され、紐づいていないものは物理削除されること。
* **手順**:
  1. 一覧画面で `PJ001`（あらかじめシードデータとして案件 `AJ001` 等が紐づいている設定）の「削除」ボタンをクリックする。
  2. 「このプロジェクトには案件が登録されているため削除できません」というエラーが表示され、削除がブロックされることを確認する。
  3. 先ほど登録した `PJ002`（案件の紐づいていないもの）の「削除」ボタンをクリックする。
  4. 確認ダイアログ等で承認後、一覧から `PJ002` が正常に消滅することを確認する。
  5. ローカルストレージ内のオブジェクト配列から `PJ002` のレコードが完全に抹消されていることをデベロッパーツール等で確認する。
