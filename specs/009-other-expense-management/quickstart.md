# Quickstart & Verification Guide: その他経費入力 (other-expense-management)

F09「その他経費入力」機能の検証シナリオ、および実行手順を示すガイド。

## 1. 開発環境の起動
```bash
# 依存パッケージのインストール（未完了の場合）
npm install

# ローカル開発サーバー（デバッグポータル）の起動
npm run dev
```
ブラウザで `http://localhost:5173/` にアクセスし、ナビゲーションから「工数実績入力」または「その他経費入力」（アサイン明細画面からの遷移など）が確認できることを検証します。

---

## 2. 受入検証シナリオ (Acceptance Verification Scenarios)

### シナリオ1: 経費一覧の確認とサマリ集計の検証 (US1)
1. アプリケーションを起動し、「アサイン契約」タブ（CaseAssignmentView）を開く。
2. `WK001` (開始: 2026-08-15) の製造原価に「5,242,000」が表示されており、粗利率が「34%」であることを確認する。
   - この原価の内訳に、その他経費として `62,000円` が含まれている（シードデータ: 50,000円 + 12,000円）。
3. アサイン明細の `WK001` 行にある「経費入力」または「その他経費」への切り替えリンクをクリックし、その他経費入力ビューを開く。
4. `WK001` に対するその他経費一覧に以下の2件が表示されていることを確認する。
   - `行No: 1`, `金額: 50,000円`, `摘要: 旅費交通費`
   - `行No: 2`, `金額: 12,000円`, `摘要: 会議費`
5. 一覧の下部またはヘッダーに合計経費として `62,000円` が表示されていることを確認する。

### シナリオ2: 経費の新規登録と自動採番の検証 (US2)
1. シナリオ1の `WK001` 経費一覧画面で、「新規経費登録」をクリックする。
2. 金額に「10,000」、摘要に「接待交際費」を入力して「登録」をクリックする。
3. 経費一覧に `行No: 3` として「接待交際費 (10,000円)」が追加されることを確認する。
4. 経費合計金額が `72,000円` に更新されることを確認する。
5. 「アサイン契約」タブに戻り、`WK001` の製造原価が `5,242,000` から `5,252,000` に増額され、粗利率が自動で再計算されることを確認する。

### シナリオ3: 経費登録エラーバリデーションの検証 (US2)
1. 経費登録フォームで、金額に `-5,000`（マイナス値）を入力して登録しようとする。
2. 画面上に「金額は0以上の整数でなければなりません」というバリデーションエラーメッセージが表示され、登録がブロックされることを確認する。
3. 摘要に 100文字を超える長い文章を入力して登録しようとした際にも、エラーが表示されてブロックされることを確認する。

### シナリオ4: 登録済みの経費の編集と物理削除の検証 (US3, US4)
1. `WK001` の `行No: 2`（会議費: 12,000円）の「編集」をクリックし、金額を「15,000円」、摘要を「会議費（弁当含む）」に変更して保存する。
2. 一覧上の合計経費が `75,000円`（接待交際費追加後の場合は `85,000円`）になることを確認する。
3. 同レコードの「削除」をクリックする。
4. 「この経費明細を削除しますか？」というブラウザの確認ダイアログが表示されることを確認し、「OK」をクリックする。
5. 一覧から該当行が消去され、合計経費が `10,000円` 減算されることを確認する。
6. 「アサイン契約」タブに戻り、`WK001` の製造原価から経費が正しく減算されて再計算されることを確認する。
