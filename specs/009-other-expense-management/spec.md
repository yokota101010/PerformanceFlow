# Feature Specification: その他経費入力 (other-expense-management)

**Feature Branch**: `009-other-expense-management`

**Created**: 2026-07-19

**Status**: Draft

**Input**: User description: "F09の開発を行う"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - その他経費の一覧表示と合計サマリ (Priority: P1) 🎯 MVP

作業契約ID（アサイン明細）ごとに発生しているその他経費の一覧、およびその合計金額を画面上に表示する。

**Why this priority**:
MVPとして最も重要な経費の可視化を担保するため。他の CRUD 機能がなくても、既存データ（初期シードデータなど）が正しく画面にロードされ、アサイン契約の製造原価と整合性が取れていることを確認できる。

**Independent Test**:
テスト用LocalStorageに初期シード（WK001に2件、WK002に1件）がロードされた状態でアプリを起動し、「その他経費入力」画面（またはアサイン明細からの経費リンク）を開いた際、指定された契約IDに紐づく経費明細が正しい金額・摘要で一覧表示され、その合計金額が正しく集計されていること。

**Acceptance Scenarios**:

1. **Given** 経費の初期シードデータ（WK001に50,000円と12,000円）が登録されているとき、
   **When** ユーザーが「作業契約ID: WK001」の経費一覧を表示したとき、
   **Then** 一覧に「旅費交通費 (50,000円)」「会議費 (12,000円)」が表示され、合計経費として「62,000円」が表示されること。
2. **Given** 経費が登録されていない作業契約（例: WK003）を選択しているとき、
   **When** ユーザーが経費一覧を表示したとき、
   **Then** 一覧は空であり、合計経費として「0円」が表示されること。

---

### User Story 2 - その他経費の新規登録 (Priority: P1)

作業契約に対して、金額と摘要を指定して新しくその他経費を登録する。

**Why this priority**:
プロジェクトで発生した新たな経費を追加して、案件全体の製造原価に即座に反映させるため。

**Independent Test**:
特定の作業契約を選択した状態で新規経費を登録した際、一意な `行No`（既存最大値 + 1）が自動採番され、LocalStorageに正しく追加保存され、一覧および合計金額が即座に再計算されて更新されること。

**Acceptance Scenarios**:

1. **Given** ユーザーが「作業契約ID: WK001」の経費一覧画面を開いているとき、
   **When** ユーザーが金額「10,000円」、摘要「接待交際費」を入力して「登録」をクリックしたとき、
   **Then** 自動的に「行No: 3」として登録され、一覧に「接待交際費 (10,000円)」が追加され、合計経費が「72,000円」に更新されること。
2. **Given** 経費登録フォームを開いているとき、
   **When** ユーザーが金額にバリデーションエラーとなる値を入力して登録しようとしたとき、
   **Then** 金額が0未満（マイナス値）の場合は「金額は0以上の整数でなければなりません」というエラーが表示され登録がブロックされること。

---

### User Story 3 - その他経費の編集・更新 (Priority: P2)

登録済みのその他経費の内容（金額・摘要）を編集変更する。

**Why this priority**:
入力ミスの修正や、後から経費金額が確定した際に変更を行えるようにするため。

**Independent Test**:
既存の経費明細の「編集」をクリックしてダイアログまたは入力フォームを開き、金額や摘要を変更して保存した際、ドメインモデルの制約に従ってデータが更新され、一覧および合計金額が再計算されて即座に反映されること。

**Acceptance Scenarios**:

1. **Given** 「作業契約ID: WK001」の「行No: 2（会議費: 12,000円）」が存在するとき、
   **When** ユーザーが金額を「15,000円」、摘要を「会議費（弁当代含む）」に変更して「保存」をクリックしたとき、
   **Then** 該当レコードが更新され、一覧の表示が変わり、合計経費が「65,000円」に再計算されること。

---

### User Story 4 - その他経費の物理削除 (Priority: P2)

不要になったその他経費データを一覧から物理削除する。

**Why this priority**:
誤って登録した経費や、キャンセルされた経費を削除し、製造原価の過剰計上を防ぐため。

**Independent Test**:
対象の経費明細の「削除」ボタンをクリックした際、確認用ダイアログ（「この経費明細を削除しますか？」）が表示され、ユーザーがOKを押した後に物理削除され、一覧および合計金額からその経費分が即座に減算されること。

**Acceptance Scenarios**:

1. **Given** 「作業契約ID: WK001」の「行No: 2（会議費: 12,000円）」が存在するとき、
   **When** ユーザーが「削除」をクリックし、確認に対して承認したとき、
   **Then** 該当レコードが物理削除され、一覧から消去され、合計経費が「50,000円」に減算更新されること。

---

### Edge Cases

- **作業契約の存在確認**: 存在しない作業契約IDに対して経費を追加しようとした場合、システムはこれをエラーとしてブロックし、不正な経費データが浮かないように制御する。
- **入力値バリデーション**:
  - 金額: 0以上の整数のみ許可。整数以外（小数点など）は入力不可。
  - 摘要: 最大100文字までに制限。

## Requirements *(mandatory)*

### ユビキタス言語辞書

| 業務上の用語 | コード内の英語表記 | 説明 |
| :--- | :--- | :--- |
| その他経費 | OtherExpense | 作業契約（アサイン明細）ごとに計上される、発注や工数加工費以外の経費（旅費など） |
| 作業契約ID | caseAssignmentId | 経費が紐づく案件作業明細（アサイン契約）の一意のID |
| 行No | lineNo | 同一の作業契約内で経費明細を一意に識別するための連番（1から開始） |
| 金額 | amount | 経費の金額（円単位） |
| 摘要 | memo | 経費の内容や名目を示す文字列 |

### Functional Requirements

- **FR-001**: システムはその他経費データを `作業契約ID` と `行No` の複合主キーで一元管理しなければならない。
- **FR-002**: 新規登録時、`行No` は指定された `作業契約ID` に属する既存データの最大 `行No` + 1 として自動的に決定・採番されなければならない（既存データがない場合は 1 とする）。
- **FR-003**: システムはその他経費の金額が変更（登録・更新・削除）された際、紐づく `作業契約ID` に紐づくその他経費の合計金額を即座に再計算し、アサイン明細管理（F06）の「製造原価」に動的に反映させなければならない（結果整合性・プル型同期）。
- **FR-004**: 金額の入力値は 0 以上の整数でなければならない。
- **FR-005**: 摘要の文字列は 100 文字以下でなければならない。

### Key Entities

- **その他経費 (OtherExpense)**:
  - `caseAssignmentId` (string, PK, FK): 対象の作業契約ID。
  - `lineNo` (number, PK): 同一作業契約内の連番。
  - `amount` (number): 経費金額。
  - `memo` (string): 経費の摘要。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがその他経費を入力・保存・削除してから、アサイン明細（F06）の「製造原価」および「粗利率」が再計算されて画面に反映されるまでの時間が 0.5秒以内であること。
- **SC-002**: システム起動時に、LocalStorage から自動で初期シードデータ 3件（WK001に2件、WK002に1件）が正しく復元され、一覧に表示されること。

## Assumptions

- ユーザーは「アサイン契約（F06）」で登録されている任意の作業契約に対して、その他経費を自由に登録できるものとする。
- その他経費の編集において、主キーである `作業契約ID` および `行No` の変更は不可とする（金額と摘要のみ編集可能）。
- 画面レイアウトは憲法の「スタンドアロンSPA」に準拠し、アサイン明細画面（CaseAssignmentView）などから直接リンクまたはタブ切り替えで「その他経費入力」画面へアクセスできる構成とする。
- データの保存はすべてブラウザの `LocalStorage` に自動保存され、次回起動時に復元される。
