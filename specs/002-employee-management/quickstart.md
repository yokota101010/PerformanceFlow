# Quickstart Validation Guide: F02 社員マスタ管理

本ドキュメントは、「F02 社員マスタ管理」機能の実装後、開発者や検証担当者が機能要件（CRUD、単価バリデーション、削除制限など）をローカル環境で検証するための手順および検証シナリオを定義する。

---

## 1. 準備と実行手順 (Setup & Execution)

### 前提条件 (Prerequisites)
* Node.js v18 以上
* npm パッケージマネージャ

### アプリケーションの起動と検証方法
以下のいずれかの方法でアプリケーションを起動し、検証を行います。

#### 方法A: スタンドアローン形式でのビルド・起動 (推奨)
1. 以下のコマンドでアセットをビルドします：
   ```bash
   npm run build
   ```
2. ビルド完了後、生成された `dist/index.html` 単一ファイルをブラウザで直接ダブルクリック（またはドラッグ＆ドロップ）して開きます。ローカルサーバーの起動は不要です。

#### 方法B: 開発用ローカルサーバーでの起動
1. 開発サーバーを起動します：
   ```bash
   npm run dev
   ```
2. 起動後、ブラウザで [http://localhost:5173](http://localhost:5173) を開きます。

#### 自動テストの実行
```bash
npm run test
```
Vitest による自動検証テストを実行し、すべてのテストスイートがパスすることを確認します。

## 2. 受入検証シナリオ (Validation Scenarios)

### シナリオ1: アプリ初回起動時の初期データ自動投入検証
* **検証内容**: 永続化ストレージ（LocalStorage）が空の状態でアプリを初めて開いた際、シードデータ（3名）が正しく自動投入され、社員IDの昇順で表示されること。
* **手順**:
  1. ブラウザのデベロッパーツールを開き、「アプリケーション（Application）」タブから `LocalStorage` のデータを全消去（クリア）する。
  2. 画面をリロードする。
  3. 社員一覧画面に、以下の社員情報が正しく表示されていることを確認する。
     - `EMP001`: トム・デマルコ (単価: 9,000)
     - `EMP002`: ロバート・マーチン (単価: 8,000)
     - `EMP003`: マーチン・ファウラー (単価: 10,000)

### シナリオ2: 新規登録とID自動採番の検証
* **検証内容**: 新しい社員が自動採番（最大値 + 1）されたIDで追加されること。また同姓同名の登録も許容されること。
* **手順**:
  1. 登録フォームにて、社員名に `デミ・ハリス`、単価に `7500` と入力し、「登録」ボタンをクリックする。
  2. 一覧画面に以下の通り追加されていることを確認する。
     - **社員ID**: `EMP004`（`EMP003` の最大値 `3` に `+1` されたもの）
     - **社員名**: `デミ・ハリス`
     - **単価**: `7,500`
  3. 再度登録フォームにて、同じ名前である `デミ・ハリス`、単価 `8000` と入力し、「登録」ボタンをクリックする。
  4. 重複エラーにならず、新しいID `EMP005` で正常に登録されることを確認する。

### シナリオ3: 入力値トリミングの検証
* **検証内容**: 入力前後のスペースが自動的に削除されること。
* **手順**:
  1. 登録フォームで、社員名に `  ジェラルド・ワインバーグ  `（前後にスペース）と入力し、単価 `8500` で登録を実行する。
  2. 登録完了後、一覧画面に表示された名称が `ジェラルド・ワインバーグ`（スペースなし）になっていることを確認する。
  3. 登録フォームで、社員名にスペースのみ（例: `  `）を入力して登録を実行する。
  4. 「社員名は必須です。」とエラーが表示され、登録できないことを確認する。

### シナリオ4: 単価バリデーションの検証
* **検証内容**: 不正な単価（空欄、負の数、文字、小数）の入力がブロックされること。
* **手順**:
  1. 登録フォームで、社員名に `テストユーザー`、単価に空欄または負の数（例: `-500`）を入力して登録を実行する。
  2. 「単価は0以上の整数で入力してください。」などのエラーが表示され、登録が拒否されることを確認する。
  3. 単価に小数（例: `1200.5`）や文字を入力した場合も同様にエラーになることを確認する。

### シナリオ5: 社員の物理削除と制限の検証
* **検証内容**: 案件工数実績データが紐づいている社員は削除が拒否され、紐づいていないものは物理削除されること。
* **手順**:
  1. 一覧画面で `EMP001`（工数実績データがすでに登録されている社員）の「削除」ボタンをクリックする。
  2. 「この社員には案件工数実績が登録されているため削除できません。」というエラー警告が表示され、削除がブロックされることを確認する。
  3. 先ほど登録した `EMP004`（実績の紐づいていない社員）の「削除」ボタンをクリックし、確認ダイアログでOKを押す。
  4. 一覧から `EMP004` が正常に消滅することを確認する。
  5. LocalStorage内のレコードから `EMP004` のデータが物理的に抹消されていることをデベロッパーツール等で確認する。
