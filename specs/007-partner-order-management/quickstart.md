# Quickstart: F07 発注（注文）管理

本ガイドは、「F07 発注管理」機能の実装後、開発者や検証担当者が主要な機能要件（一覧、新規登録、金額・工数の自動計算、所属会社バリデーション、カスケード削除など）をローカル環境で手動検証するための手順および検証シナリオを定義する。

---

## 準備

検証を開始する前に、以下のコマンドを実行し開発環境を起動する。

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```

起動後、ブラウザで [http://localhost:5173/](http://localhost:5173/) を開く。
ナビゲーションヘッダーより「発注管理」のタブ（F07）を選択する。

---

## 検証シナリオ

### シナリオ1: 初期シードデータの表示検証 (US1)

* **目的**: 初回起動時にシードデータが正しく読み込まれ、合計工数・発注額が自動計算されて一覧表示されることを確認する。
* **手順**:
  1. 「発注管理」タブを開く。
  2. テーブル上に `ORD001` から `ORD006` までの発注データが表示されていることを確認する。
  3. 各行の「合計工数」および「合計発注額」が以下のように正しく表示されていることを確認する。
     - `ORD001`: 合計工数 `1.3`、合計発注額 `1,150,000円`
     - `ORD005`: 合計工数 `1.6`、合計発注額 `1,210,000円`
  4. `ORD001` 行の「詳細」ボタンをクリックする。
  5. 画面下部の「注文明細」テーブルに以下の2行が表示されることを確認する。
     - `MEM001`: 工数 `0.8`、単価 `1,000,000円`、発注額 `800,000円`
     - `MEM002`: 工数 `0.5`、単価 `700,000円`、発注額 `350,000円`

### シナリオ2: 新規発注の登録と年月制約の検証 (US2)

* **目的**: 新しい発注データの登録時に、自動採番（`ORD007`）されること、および作業契約期間外の年月入力がエラーブロックされることを確認する。
* **手順**:
  1. 発注登録フォームで以下を選択する。
     - **作業契約**: `WK001` (対象期間: 2026-08-15 〜 2026-09-30 と仮定)
     - **発注先**: `BP001` (Aソフト開発支援の委託先)
  2. **年月** に期間外の `2026-07` (2026-07-01) を入力し、「登録」ボタンをクリックする。
  3. 画面上に `選択された年月は作業契約の対象期間外です。` というエラー警告が表示され、登録が拒否されることを確認する。
  4. 続いて **年月** に期間内の `2026-08` を入力し、再度「登録」をクリックする。
  5. エラーが消え、一覧に新しく `ORD007` のレコードが「合計工数 0.0」「合計発注額 0円」として追加されることを確認する。

### シナリオ3: 明細の追加と所属会社バリデーションの検証 (US2)

* **目的**: 注文明細に要員を追加する際、発注先企業と一致しない他社所属の要員登録がブロックされることを確認する。
* **手順**:
  1. 前のステップで追加した `ORD007`（発注先: `BP001`）の「編集/明細追加」をクリックする。
  2. 明細行追加フォームの要員セレクトボックスにおいて、`BP002`（別会社）所属の要員 `MEM003` が選択肢から除外されているか、あるいは選択して登録しようとした際に `要員の所属会社と発注先が一致しません。` エラーで保存がブロックされることを確認する。
  3. `BP001` 所属の要員 `MEM001` を選択し、発注工数に `0.8` 人月を入力して明細行を追加・保存する。
  4. 一覧で `ORD007` の合計工数が `0.8`、合計発注額が `800,000円` (MEM001単価100万×0.8) に自動計算されて表示されることを確認する。

### シナリオ4: 工数の更新と自動計算の検証 (US3)

* **目的**: 稼働工数の変更が、発注額および合計工数・合計発注額にリアルタイムで反映され保存されることを確認する。
* **手順**:
  1. `ORD001` の「編集」をクリックする。
  2. 明細行テーブル上の `MEM001` の発注工数を `0.8` から `0.5` に書き換え、「保存」をクリックする。
  3. `ORD001` の合計工数が `1.0` 人月、合計発注額が `850,000円`（0.5*100万 + 0.5*70万）に更新されていることを確認する。

### シナリオ5: 発注データの物理削除とカスケード削除の検証 (US4)

* **目的**: 不要な発注データを削除した際、紐づく明細もカスケード削除され、一覧から消去されることを確認する。
* **手順**:
  1. `ORD007` の「削除」ボタンをクリックする。
  2. 警告ダイアログの「OK」をクリックする。
  3. `ORD007` が一覧テーブルから消滅することを確認する。
  4. 画面をリロードし、LocalStorage からもデータが消え、再表示されないことを確認する。
